# Contributing to Deluge

Thank you for your interest in improving Deluge! We welcome contributions from the open-source community, particularly those with experience in GIS, FastAPI, and React.

## How to Contribute

1. **Fork the Repository**: Create your own fork and branch off `main`.
2. **Discuss Major Changes**: For significant architectural changes, please open an Issue first to discuss it with the maintainers.
3. **Follow Coding Standards**: Ensure your code adheres to our [Coding Standards](file:///c:/Users/nihal/OneDrive/Desktop/Deluge/documentation/coding-standards.md).
4. **No "Simulation Game" Logic**: Deluge is an Emergency Operations Platform. Features that treat the platform like a game (e.g., arbitrary weather manipulation buttons, gamified scoring) will be rejected. 
5. **Submit a Pull Request**: Keep PRs focused on a single feature or bug fix. Include a detailed description of what you changed and why.

## Reporting Bugs

Please open an issue providing:
- A clear, descriptive title.
- Steps to reproduce the issue.
- Expected vs. actual behavior.
- Context (OS, Node version, Python version).
